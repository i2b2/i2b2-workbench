i2b2 Query Tool View

The Query Tool view is designed to simplify the process of retrieving information from the database associated to the i2b2 Workbench.

 

Introduction

This section describes the layout and contents of this view

 

Layout of the Query Tool View

The Query Tool view is used to define and run a “select query” that will retrieve information from the database associated to the i2b2 Workbench.

 

 

 

The Query Tool view can be broken down into several sections which are defined below.

 

Query Name

The name of a query will be assigned when it is run and will display in the Query Name field. 

The only time a query name will appear prior to running the query is when you drag a previous query from either Previous Query or Workplace views and drop it into the Query Name field.

 

 

 

Clear Button

The Clear button () located next to the Query Name field will reset the view to the default; this includes deleting the concepts listed in the panel and removing any constraints that have been defined. 

 

Output Selection

In the output selection area you will define the Query Timing and select what to return when running the query.

 

 

Analysis Types

Select what to return when running the query. There are several options available for selection and they can be selected in any combination.

 

 

 

Patient list

Provides a set of patients who met the search criteria.

Event list

Provides a set of encounters for those patients who met the search criteria.

Number of patients

Provides the total number of patients who met the search criteria.

Gender patient breakdown

Provides a breakdown of the patients by their gender.

Vital Status patient breakdown

Provides a breakdown of the patients by their vital status.

Race patient breakdown

Provides a breakdown of the patients by their race.

Age patient breakdown

Provides a breakdown of the patients by their age

Timeline

Provides a set of patients who met the defined criteria and automatically generates a timeline in the timeline view.

 

 

Note: The only difference between the Patient list and Timeline options is the Patient list will not generate a timeline when the query has completed running.  The Timeline option will automatically generate a timeline in the timeline view.

 

 

Query Timing

The Query Timing is defined on the query level and depending on what is selected will set the default for the panel timing.

 

 

 

Panel

A panel is the section of the Query Tool view that contains information about a group of terms.  The panel can be broken down into the following sections.

 

 

 

Group Heading

The heading is a label to easily identify the collection of terms. Each heading will contain “Group” followed by a sequential number.

Example: Group 1, Group 2, Group 3, etc.

 

Selection Criteria

This area of the panel is used to define date selections and other constraints.

 

Panel Timing

This panel timing defines the constraint on the panel level and uses the patient’s visit information in determining eligibility.

 

Group List

This section contains the terms that will be used during the process of querying the database.

 

Information Box

The content in the information box will change depending on whether or not a term is listed in the panel / group.

a.)    No Term(s) listed

Drag terms from Navigate, Find and Workplace into this group

 

b.)    Terms(s) listed

The terms of this group are joined then intersected with other groups

 

 

Toolbar

The toolbar is the bottom section of the Query Tool view and it contains several buttons that perform various actions.

 

 

 

Patients Returned Box

In the bottom right corner of the query view is the Patients Returned box.  Once a query has been run, the number of patients meeting the search criteria will appear in this box.  This is a display only box.

 

 

 

Get Everyone Checkbox

In the bottom left corner of the query view is the Get Everyone checkbox.  A check in this box will ignore any search criteria you may have entered and return a list of all the patients in the database.

 

 

Run Query / Cancel / Get All Patients Button

On the toolbar for the Query Tool view there is a single button for running and canceling a query as well as getting all patients. Depending on the circumstances defined below the label on this button and its associated action will change to one of the following: (1) Run Query (2) Cancel and (3) Get All Patients.

 

Run Query Button

Clicking on the button when the Run Query Above label is displayed will run your query with the defined search criteria.  This is the default option.

 

 

Cancel Button

Once you start running a query, the label will change to Cancel.  Clicking on the button when it says Cancel will cancel the query and free up the workbench so that you can continue to do other things. 

 

Note: Clicking on Cancel in the Query Tool will only cancel the workbench piece of the process.  To cancel the server side you need to go to Previous Query, right click on the query name and select cancel from the menu.

 

 

Get All Patients Button

A check in the Get Everyone checkbox will change the label to Get All Patients.  Clicking on the button when it says Get All Patients will run a query to retrieve all patients in the database.  Any selection criteria you entered will be ignored.

 

 

 

Options

Users can define how many terms to display when expanding a folder in the panel and how log to wait before a query will time out. These settings are defined in the Query Tool Options, which can be accessed by clicking on the Options button ().
