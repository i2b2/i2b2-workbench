Introduction

This section describes the layout and contents of the Previous Query view.

 

Layout of the Previous Query View

A previous query () is created automatically when a query is run from the Query Tool.  The query is stored and displayed in the Previous Query View. These previous queries can be used to run new queries or to easily populate other views such as the timeline view.

 

Figure 1         Previous Query View

 

 

The query in the previous query is comprised of several levels that can be seen by expanding each level of the tree.  These levels are outlined below:

 

Previous Query Name

The name of the previous query is the top level of the hierarchal tree and is comprised of these three components.

 

1.       Query Name – the name given at the time the query was created. The default name contains an abbreviated description and the time the query was run.  

2.       Date – the date the query was created.

3.       User Id – the user who created (ran) the query.

 

Figure 2         Previous Query Naming Components

 

 

Note: unless you are a manager you will only see your queries.

 

Results Folder

The results folder () contains the results of the query.  To see the query results simply expand the view by clicking on the plus sign () located next to the result name.

 

Results

The results associated with a previous query are determined by the output options (Analysis types) that are selected when the query is run in the Query Tool view.   

 

Patient List

The Patient List () appears if either Patient List or Timeline was checked at the time the query was run.  The list contains all the patients who met the defined search criteria. Included in the list are the Patient ID and basic demographic information.

 

Figure 3         Patient List

 

 

Note: If both Patient list and Timeline are checked when running the query there will be only one Patient list associated with the previous query and displayed in the view.

 

The only other way to generate a patient list is to check off Get Everyone when running a query.  The query will return only a list of patients and no concepts will be associated with the previous query.

 

Figure 4         All Patients Previous Query

 

 

Number of Patients

The Number of Patients () appears if Number of Patients was checked at the time the query was run.  The results are a total count for those patients who met the defined criteria.  The actual number is displayed in the tool tip, which can be seen by hovering over it. 

 

Figure 5         Number of Patients

 

 

Gender Patient Breakdown

The Gender patient breakdown () appears if Gender patient breakdown was checked at the time the query was run.  The results are a breakdown by gender for those patients who met the defined criteria.

 

Figure 6         Gender Patient Breakdown

 

 

Vital Status Patient Breakdown

The Vital Status patient breakdown () appears if Vital Status patient breakdown was checked at the time the query was run.  The results are a breakdown by vital status for those patients who met the defined criteria.

 

Figure 7         Vital Status Patient Breakdown

 

 

Race Patient Breakdown

The Race patient breakdown () appears if Race patient breakdown was checked at the time the query was run.  The results are a breakdown by race for those patients who met the defined criteria.

 

Figure 8         Race Patient Breakdown

 

 

Age Patient Breakdown

The Age patient breakdown () appears if Age patient breakdown was checked at the time the query was run.  The results are a breakdown by age for those patients who met the defined criteria.

 

Figure 9         Age Patient Breakdown

 

 

 

Navigation

Rename a Previous Query

Users can rename a previous query from the list by selecting Rename from the popup menu.

 

1.       Using the right mouse button, click on the Previous Query to be renamed.

2.       A popup menu will open.

 

Figure 10       Popup Menu

 

 

3.       Select Rename from the list.

 

Figure 11       Rename Selection

 

 

4.       The rename query dialog box will open.

 

Figure 12       Rename Query Dialog

 

 

5.       Enter the new name of the previous query and click on the OK button.

6.       The previous query will now display in the view with the new name.

 

Note: When dragging a previous query to another view the new name will appear when it is dropped.

 

 

Delete/Remove a Previous Query

Users can remove a previous query from the list by selecting Delete from the popup menu.

 

1.       Using the right mouse button, click on the Previous Query to be deleted.

2.       A popup menu will open.

3.       Select Delete from the list.

 

Figure 13       Delete Selection

 

 

Warning: Once you click on Delete, the previous query will be removed. Therefore you need to make sure you want to remove it from the list.

 

 

Refresh the List

Users can refresh the list of previous queries by selecting Refresh All from the popup menu.

 

1.       Using the right mouse button, click on any of the previous queries in the list.

2.       A popup menu will open.

3.       Select Refresh All from the list.

 

Figure 14       Refresh All Selection

 

 

4.       The list of previous query and their status will be refreshed.

 

 

Cancel a Previous Query

Cancelling a query is a two step process.  The first step is to cancel it while it is running in the query tool view.  This is done be clicking on the cancel button.  The first step only cancels the query on the workbench side.  The query will continue to run on the server.  From this point you have two options: (1) Do nothing and let the query continue to retrieve data on the server or (2) Cancel the query on the server side which is done by cancelling the previous query in the previous query view.

 

In previous query view “INCOMPLETE” will appear next to the results folder when a query is still running on the server.  In addition you will see “QUEUED” next to the results listed in the results folder.

 

Figure 15       Incomplete Query

 

 

1.       Click on the plus sign () next to the name of the previous query that is to be cancelled.

2.       Using the right mouse button, click on the name of the previous query.

3.       A popup menu will open.

4.       Select Cancel from the list.

 

Figure 16       Cancel Selection

 

 

5.       A message window will open stating the query is finished.

 

Figure 17       Query Finished Message

 

 

6.       Click on the OK button to close the message window.

7.       The CANCELLED status will now appear next to the result folder.

 

Figure 18       Cancelled Status

 

 

Display Options

Users can define how many previous queries to display and the order in which they will appear in the list.  The display options are set in the Previous Queries Options Dialog window, which can be accessed by clicking on the Options () button.

 

Figure 19       Display Options

 

 

Maximum to Display

Queries to Display

The Maximum number of queries to be displayed defines how many previous queries will display in the Previous Query View. Once the maximum number is reached the oldest query will no longer appear in the view.

 

Recommendation: Use the Workplace View to save those previous queries that will be used often. In addition to being more convenient it will also prevent you from loosing a common query if the maximum number to display has been reached.

 

Patients to Display

The Maximum number of patients to be displayed is used when the system is retrieving the list of patients to display in the Previous Query View.  Once the maximum number is reached a message will appear informing you that you have reached the maximum defined and that continuing to retrieve all the patients in the list may affect performance. You will have the option to continue or to cancel the action.

 

Figure 20       Patient Count Message

 

 

A Yes selection will close the window and display “Working…” under the patient list.  Once the list finishes compiling it will display all the patients in the set.

 

Figure 21       Working Status

 

 

A No selection will close the window and display “Over maximum number of patients” under the patient list name.

 

Figure 22       Over Maximum Number of Patients

 

 

Order of Display

Sort by Time

Selecting sort by time when the query was created will sort the queries in the Previous Query View by the date and time the query was created.  Once you select to sort by time, you need to define whether or not it will be in Ascending or Descending order.

Ascending: displays the oldest query first.

Descending: displays the newest query first.

 

Note: Renaming a query will not affect the sort order as the date and time is stored with the previous query.

 

Sort by Query Name

Selecting sort by query name will sort the queries in the Previous Query View alphabetically by the name of the query.  Once you select to sort by name, you need to define whether or not it will be in Ascending or Descending order.

Ascending: queries are displayed in alphabetical order with the beginning of the alphabet at the beginning of the list. (A to Z)

Descending: queries are displayed in alphabetical order with the beginning of the alphabet at the end of the list.  (Z to A)

 

 

 

Using a Previous Query

This section describes how to use a previous query.

 

Draggable Items

Different parts of the previous query can be dragged and dropped (copied) from the Previous Query view to other views in the i2b2 Workbench.

 

1.       Previous Query

Associated with each previous query () is the concepts used to run the query and the results. A previous query can be added to other views by dragging the previous query name from Previous Query view to one of the following views.

 

a.)    Analysis View

§         Drop in the Analyses section.

 

b.)    Query Tool View

§         Drop in the Query Name field.

 

c.)     Timeline View; Create model for Timeline Tab

§         Drop in the concept list section.

 

d.)    Workplace View

§         Drop in the user’s folder or the shared folder.

 

2.       Patient list

A specific set of patients can be added to other views by dragging the Patient list () from Previous Query view to one of the following views.

 

a.)    Timeline View; Create model for Timeline Tab

§         Drop in the Patient Set field.

 

b.)    Workplace View

§         Drop in the user’s folder or the shared folder

 

3.       Individual Patient

An individual patient can be added to the Workplace View by dragging the patient () from Previous Query view.

 

4.       Patient Count

The patient count can be added to the Workplace View by dragging number of patients () from Previous Query view.

 

 

Add to Query Tool View

A previous query can be added to the query tool view by simply dragging the query name from Previous Query view.

 

Add Previous Query To Query Tool View

1.       Highlight the previous query by clicking on the name of the query.

2.       While holding the left mouse button down, drag the previous query name to the Query Tool View.

3.       Drop the previous query into the Query Name.

 

Figure 23       Dropping Previous Query in Query Tool View

 

 

4.       The concept(s) associated with the previous query will display in the appropriate group(s) and the name of the query will appear at Query Name.

 

Figure 24       Previous Query Added to Query Tool View

 

 

Add to Workplace View

Information in the workplace is related to the most common concepts and queries that an individual uses and in essence becomes their personal workplace.

 

Figure 25       Workplace View

 

 

Add Previous Query To Workplace View

1.       Highlight the previous query by clicking on the name of the query.

2.       While holding the left mouse button down, drag the previous query name to the Workplace View.

3.       Drop the previous query into either your workplace folder or the shared folder.

4.       The previous query will now appear in the workplace with the same name.

 

Add Patient List To Workplace View

1.       Click on the plus sign () next to the name of the previous query that contains the patient list.

2.       Click on the plus sign () next to the results folder.

3.       Click on patient list ().

4.       While holding the left mouse button down, drag the patient list to the Workplace View.

5.       Drop the patient list into either your workplace folder or the shared folder.

6.       The patient list will now appear in the workplace.

 

Add A Patient To Workplace View

1.       Click on the plus sign () next to the name of the previous query that contains the patient list.

2.       Click on the plus sign () next to the results folder.

3.       Click on the plus sign () next to patient list.

4.       Click on the patient () to be copied.

5.       While holding the left mouse button down, drag the patient to the Workplace View.

6.       Drop the patient list into either your workplace folder or the shared folder.

7.       The patient will now appear in the workplace.

 

Add Number of Patients To Workplace View

1.       Click on the plus sign () next to the name of the previous query that contains the number of patients result.

2.       Click on the plus sign () next to the results folder.

3.       Click on number of patients ().

4.       While holding the left mouse button down, drag the number of patients to the Workplace View.

5.       Drop the number of patients into either your workplace folder or the shared folder.

6.       The number of patients will now appear in the workplace as patient count ().

 

 

Add to Timeline View

The timeline view displays the information in a graphical representation of a chronological sequence of events.

 

Figure 26       Timeline View

 

 

Add Previous Query To Timeline View

1.       Highlight the previous query by clicking on the name of the query.

2.       While holding the left mouse button down, drag the previous query name to the Timeline View.

3.       Drop the previous query into the Query Name field on the Create model for timeline tab.

4.       The concepts and patient list associated with the previous query will now appear in the appropriate sections of the Timeline view.

 

Add Patient List To Timeline View

1.       Click on the plus sign () next to the name of the previous query that contains the patient list.

2.       Click on the plus sign () next to the results folder.

3.       Click on patient list ().

4.       While holding the left mouse button down, drag the patient list to the Timeline View.

5.       Drop the patient list into the Patient Set field located on the Create model for timeline tab.

6.       The patient set field will now display the name of the patient list.

 

 

Add to Analysis View

The Analysis view is designed to provide the visualization of analysis results of queries that users run using the i2b2 Workbench.

 

Figure 27       Analysis View

 

 

Add Previous Query To Analysis View

1.       Highlight the previous query by clicking on the name of the query.

2.       While holding the left mouse button down, drag the previous query name to the Analysis View.

3.       Drop the previous query into Graphic Analyses section of the view.

4.       The analysis results associated with the previous query will now appear in the Analysis View.
