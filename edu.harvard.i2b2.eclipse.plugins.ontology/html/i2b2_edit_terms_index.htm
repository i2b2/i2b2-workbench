i2b2 Edit Terms View

The Edit Terms View is designed to enable users to easily add custom metadata into the Ontology Management Cell of the i2b2 Hive. The concepts added via this view will be available for use in many of the i2b2 views, such as Query Tool and Timeline.

Introduction

This section describes the layout and contents of this view.

 

Layout of the Edit Terms View

The Edit Terms view displays the concepts listed in the i2b2 metadata table as well as any concepts added to the custom metadata table.  These concepts are grouped (categorized) with other similar items.  These groupings are displayed in a hierarchal tree that can be expanded to view all the concepts in the associated category.

 

Figure 1         Edit Terms View

 

 

The overall appearance of the Edit Terms view is very similar to the Navigate Terms view. The main difference is the icons for the containers, folders and items are slightly different. Those items that can be edited will contain a pencil in the icon and those that can not be edited will contain a lock.

 

As stated above, the Edit Terms view is hierarchical display of the concepts in the metadata table.  There are several main branches or components of the tree that displays.

 

Table Name

The highest level is the name of the metadata table that stores the vocabulary.

 

Containers

A container () contains other folders and/or concepts.  It can be expanded and collapsed by clicking on the plus or minus sign next to it.  In the Edit Terms view a container will be one of the following:

1.       Editable (), which means additional folders and/or concepts can be added to it.

2.       Non-Editable (), which means the container is locked and additional folders and/or concepts can not be added to it.

 

Folders

A folder () contains other folders and/or concepts.  It can be expanded and collapsed by clicking on the plus or minus sign next to it.  In the Edit Terms view a folder will be one of the following:

1.       Editable (), which means additional folders and/or concepts can be added to it.

2.       Non-Editable (), which means the container is locked and additional folders and/or concepts can not be added to it.

 

Item

The item () is the lowest level of the hierarchy and cannot be expanded further.

 

 

 

Navigation

This section contains information about expanding and collapsing folders and items on the pop-up menu.

 

Expand/Collapse a Folder

Users can expand or collapse a folder by clicking on the plus sign () or negative sign () next to the folder.

 

Expand (Open) a Folder

1.       Click on the plus sign () next to the folder in Edit Terms view.

2.       The folder will open and all the items in the folder will display.

 

Collapse (Close) a Folder

1.       Click on the negative sign () next to the folder in Edit Terms view.

2.       The folder will close and all the items in the folder will no longer appear in the view.

Adding Terms

The Edit Terms view has an insertion wizard that is designed to assist users with the process of adding a new container, folder or item to the appropriate tables.

 

Using the Insertion Wizard

The following steps outline the process for starting the wizard.

 

1.       Right click on the container or folder you want to add a term to.

2.       Highlight New on the pop-up menu.

3.       The submenu displaying the types of terms that can be added will display.

4.       Select the appropriate type of term you want to add.

5.       The first page of the wizard will open.

Page Five: Enumeration or Value Range Metadata Settings

The data type defined on page four will determine whether or not the fifth page of the Insertion Wizard will be the Enumeration Metadata Settings page or the Value Range Metadata Settings page.

 

Enumeration Metadata Settings

The fifth page of the Insertion Wizard allows users to enter text values for the term.

 

Example:

For the term UA - Protein the enumerated values would be:

        Positive

        Negative

        Pos

        Neg

        Test Not Performed

        TNP

 

Value Range Metadata Settings

The fifth page of the Insertion Wizard allows users to enter the range of numerical values for the term.

 

       The ranges entered on this page are used when the term is added to the Timeline view it does not determine the high/low flags associated with the observation.

 

The following outlines each of the fields on this page.

 

Field

Description

Example

Low end of low value range

The lowest numerical value in the Low Range

1

High end of low value range

The highest numerical value in the Low Range

134

Low end of high value range

The lowest numerical value in the High Range

146

High end of value range

The highest numerical value in the High Range

300

Low end of toxic value range

The lowest numerical value in the Toxic Range

301

High end of toxic value range

The highest numerical value in the Toxic Range

500

 

 

The value ranges are very similar to the reference ranges you see on a lab report.  For instance, for a standard sodium test the reference range may be 135  145.  Anything below 135 is considered low and anything above 145 is considered high. In this case the High end of low value range would be 134 and the Low end of high value range would be 146.

 

Not all numerical terms will have a toxic value range which is why these fields are optional. You may choose to use these fields for those tests that do have toxic ranges associated. Example: Digoxin level.  The therapeutic range would be 0.8 to 2.0 ng/ml and the toxic level would be >2.4 ng/ml

Page Seven: Ontology Term Verification

The last page of the wizard is a summary of the settings defined on the previous pages. If you need to change settings simply click on the Back button until you are at the appropriate page.  Once you have verified everything click on the Finish button to commit the term to the database.

 

 

Update and Synchronize Terms

The final step in adding a term is to synchronize the custom metadata table with the concept dimension table. The following outlines the steps in this process.

 

1.       Click on the synchronize icon () at the top of the view.

       The arrows on the synchronize icon will turn red () when an update or synchronization is needed.

 

2.       The Update Ontology window will open.

 

Figure 5         Update Ontology window

 

 

3.       Click on the OK button to start the process.

 

 

 

Editing Terms

The Edit Terms view has an edit term wizard that is designed to assist users with the process of editing a container, folder or item.

 

Using the Edit Term Wizard

The following steps outline the process for starting the wizard.

 

1.       Right click on the container, folder or item you want to edit.

2.       Select Edit on the pop-up menu.

3.       The first page of the edit term wizard will open.

4.       Use the Next button to advance to the other pages.

 

       For additional information on each of the pages please see the section labeled Adding Terms.

 

5.       Once your edits have been made click on the Finish button.

 

       The Finish button will not be activated until you have reached the last page of the wizard.
