Page One: Settings

The following outlines each of the fields on the first page of the Insertion Wizard.

 

Field

Description

Example

Container Name

 

The descriptive text value for the container. This information is displayed to the user in both navigate and find terms views.

Annotator Terms

Tooltip

The tooltip that will appear to the users when they hover over the term.

Custom Metadata \ Annotator Terms

Base code

A code that represents the term in the concept dimension table.

The first field contains the prefix which represents the coding system.  (Ex.: LOINC, ICD-9, etc.)

The second field is a text box where you can enter the code for this concept.

        Containers do not have base codes therefore this field is not available.

Symbol Name

The symbol name is appended to the path to create the full path name.

This field cannot contain duplicates as the full path name needs to be unique.

The name will be truncated to 32 characters.

Annotator Terms

Sourcesystem code

A coded value for the source system from which the term was loaded or derived.  The system will default the following when adding a new term from the Edit View:

userID_manualentry

 

        Note: userID is replaced with the id of the user logged on to the i2b2 Workbench.

demo_manualentry

Comment

An optional field to store miscellaneous comments.

Container or terms to be used with the annotator view.

Semantic type

Denotes the type of term. i.e. Document or Laboratory term

Example:

Doc - Radiology Report, patient notes.

Lab – Sodium, Calcium, UA

Enter value information

Determines whether or not you want to enter value data for the term.

A checkmark indicates that you want to enter value data.  When this is checked additional pages will be made available in the wizard.

        Note: When the semantic type is “Lab” this field will automatically be checked.
