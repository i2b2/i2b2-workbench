Value Constraint

As stated earlier, not all concepts have values.  The text that displays in the Value constraint column will assist users in knowing whether or not a concept can have a value constraint.

 

“Not Applicable” will appear in the Value Constraint column for those concepts that do not have values.

 

Figure 25       Value Constraint Not Applicable

 

 

“All values” will appear in the Value Constraint column for those concepts that do have values associated and have no constraints defined.

 

Figure 26       Value Constraint All Values

 

 

Flag Value

Flags are defined in the source system when the value is entered.  The following are some examples that can be found in the i2b2 Demo database.

§         High

§         Low

§         Abnormal

 

Setting the Value Constraint (Flag)

a.       Highlight the concept you want to set a value constraint.

b.      Double click in the Value Constraint column for that concept.

c.       A new window will open.

 

Figure 27       Value Constraint Window (Flag)

 

 

d.      Click the radial dial next to By high/low flag.

e.      A new field with a drop down selection box will appear.

 

Figure 28       Value Constraint (Flag)

 

 

f.        Click on the arrow to change the selection or accept the default.

g.       Click on the OK button.

h.      The constraint value entered will now appear in the Value Constraint column.

 

Figure 29       Value Constraint Set (Flag)

 

 

 

Text Value

The text is defined in the source system when the value is entered.  The following are some examples that can be found in the i2b2 Demo database.

§         Test Not Performed

§         Borderline

§         Positive

 

Setting the Value Constraint (Text)

1.       Highlight the concept you want to set a value constraint

2.       Double click in the Value Constraint column for that concept.

3.       A new window will open.

 

Figure 30       Value Constraint Window (Text)

 

 

4.       Click the radial dial next to By text value.

5.       The text value options will now become available to select.  Click on the box next to the values you want to include.

 

Figure 31       Value Constraint (Text)

 

 

6.       Click on the OK button.

7.       The constraint value entered will now appear in the Constraint Value column.

 

Figure 32       Value Constraint Set (Text Value)

 

 

 

Numeric Value

The numeric value is defined in the source system when the result is entered.  The following are some example formats that can be found in the i2b2 Demo database.

§         200

§         1.3

 

Unlike flag and text values, numeric values use operators, such as greater than or less than when setting the constraints. The following is a listing of operators currently in use.

 

Operator

Description/Use

Less Than (<)

Any value less than the number entered.

Less Than or Equal To (<=)

Any value less than or equal to the number entered (inclusive).

Equal To (=)

An exact match to the number entered.

Between

Any value in the range of numbers entered.

Greater Than (>)

Any value greater than the number entered.

Greater Than or Equal To (>=)

Any value greater than or equal to the number entered (inclusive).

 

 

Setting the Value Constraint (Numeric)

1.       Highlight the concept you want to set a value constraint.

2.       Double click in the Value Constraint column for that concept.

3.       A new window will open.

 

Figure 33       Value Constraint Window (Numeric)

 

 

4.       Click the radial dial next to By numeric value. 

5.       Two new fields, one with a drop down selection box will appear.

 

Figure 34       Value Constraint (Numeric)

 

 

6.       Click on the Drop-down arrow to change the operator or accept the default.

7.       Enter a numerical value at the Please enter value field.

8.       Click on the OK button.

9.       The constraint value entered will now appear in the Constraint Value column.

 

Figure 35       Value Constraint Set (Numeric)
