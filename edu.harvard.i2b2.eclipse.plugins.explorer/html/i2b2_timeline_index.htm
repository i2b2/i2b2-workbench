Introduction

The timeline view is comprised of two tabs; (1) Create model for Timeline and (2) Render of a Timeline.  This section describes the layout and contents of both tabs.

 

Layout of the Create model for Timeline tab

The concepts to be used when rendering the timeline are defined in the Create model for Timeline tab.  Users can add individual or groups of concepts.  In addition to adding concepts, users can define value constraints for each concept and set the height and color of the ticks that appear on the timeline.

 

Figure 1         Layout of the Create model for Timeline tab

 

 

Query Name

The name of the query will appear when you drag a previous query from either Previous Query or Workplace Views and drop it into the Query Name field.

 

Panel Name

The name of the group template will appear when you drag a group template from Query tool or Workplace Views and drop it into the Panel Name field.

 

Concept List

This section contains the list of concepts to be used when rendering the timeline.  The list may be manually setup by the user or populated when dragging a previous query or group template into the view.  The following table describes each column:

 

Column Name

Description / Use

Row #

Identifies order in which the concept will display in the timeline.

Name of Terms

The name of the concept.

Date Constraint

Limit the data to a user defined date range for a particular concept.

Value Constraint

Limit the data to a user defined range of values for a particular concept.

Shape

The height of the tick on the timeline.

Color

The color of the tick on the timeline.

 

 

Deletion Buttons

Remove concepts from the list.

 

Move Buttons

Change the order of concepts on the list.  This order is also reflected in the timeline when it is generated.

 

Display Options

Select what to display when viewing the timeline. Currently there are two options.

 

Display Concepts with No Data

A check in the box next to Display concepts with no data means all concepts listed will display in the timeline even if an event for that concept has not occurred.  No check mark means only the concepts that have an event will display in the timeline.

 

Display Patient Demographics

A check in the box next to Display patient demographics means the patient’s demographic information will display.  No check mark means only the patient id will display.

 

 

Patient Set

In the bottom left corner of the timeline view is the Patient Set box.  This box will display the number of patient’s in the set that will be used when rendering the timeline.

 

 

Layout of the Render a Timeline Tab

The render a timeline tab will use the information entered on the Create model for Timeline tab.  If data has not been entered, a warning will appear and the user needs to go back and add the required information.

 

Figure 2         Layout of the Render a Timeline tab

 

 

Timeline

Graphical display of chronological events based on the concepts identified on the Create model for Timeline tab.

 

Zoom In/Out Arrows

The arrows at the bottom of the timeline allow users to zoom in and out of the timeline.

 

Increment Settings

 

Start

Defines what record number the timeline should begin the display.

Increment

Defines how many to advance when the Next and Previous buttons are clicked on.

 

 

Previous and Next Buttons

Next button moves the timeline forward to the next group of patients.

Previous button moves the timeline backwards to the previous group of patients.

 

 

 

Creating a Timeline

This section describes the process of adding items to the query tool, setting constraints and setting the display options for the timeline.

Concepts () can be defined as terms, results, providers, etc.  They can be added as a group (folder/subfolder ()) or individually by dragging the item(s) from Navigate Terms, Find Terms, and Workplace views.

Delete/Remove

There are two options for removing a concept from the list of concepts to be used. Users can (1) delete the entire listing of concepts, or (2) delete a single concept from the list.

 

Delete All Concepts

1.       Click on the Delete All button located under the list of concepts.

2.       The list of concepts will be cleared.

 

Warning: Once you click on the Delete All button, all the concepts on the list and their constraints will be removed, therefore you need to make sure you want to delete every one.

 

Delete Single Concept

1.       Click on the concept in the list that you want to remove.

3.       Once the concept is highlighted, click on the Delete From List button.

4.       The concept will be removed from the list.

 

 

Move Concepts

Each item in the list of concepts has a row # associated to it.  This row # is used to determine the order in which the concepts will appear in the timeline when it is rendered.  Users have the option of moving concepts to a new row before the timeline is created.  The system will automatically update the row #.

 

Move Concept Up

1.       Highlight the concept to be moved.

2.       Click on the Move Up button.

3.       Continue to click on the move up button until the concept is in the desired location.

 

Move Concept Down

1.       Highlight the concept to be moved.

2.       Click on the Move Down button.

3.       Continue to click on the move down button until the concept is in the desired location.

 

Put Concepts in Order

1.       Double click in the row # column next to the concept to be moved.

2.       Enter the new row #.

3.       Click on the Put In Order button.

4.       The row #’s for the remaining concepts will be updated and placed in the correct sequential order.

 

 

Setting Constraints

There are basically two categories of constraints that can be defined in the timeline view. One constraint is by the date and the other is by the value associated to the constraint.  Not all concepts have values. For instance, diagnosis of Acute Myocardial Infarction does not have a value but the tests used to make the diagnosis do have values (high, low, numeric, etc.). 

 

Date Constraint

Users can define a date range to be used when rendering a timeline. This date range can be in any one of the following combinations:

SPECIFIED DATE RANGE: FROM AND TO DATES DEFINED

Only data with activity within the defined date range will be used.

Example:

Concept: Acute Myocardial Infarction

From date: 01/01/2001

To date: 12/31/2005

In this example, if a patient has a diagnosis of Acute Myocardial Infarction (MI) entered on the record from 01/01/2001 to 12/31/2005 then their data will be included. If the patient does have an MI diagnosis but it was entered prior to 01/01/2001 or after 12/31/2005 then their information will not be included.

 

SPECIFIC START DATE: ONLY THE FROM DATE IS DEFINED

Data from the defined start date to the latest date available. No end date defined.

Example:

Concept: Acute Myocardial Infarction

From date: 01/01/2001

To date:

In this example, if a patient has a diagnosis of Acute Myocardial Infarction (MI) entered on the record any time on or after 01/01/2001 will be included.  If the diagnosis was entered prior to 01/01/2001 then the data will not be included.

 

SPECIFIC END DATE: ONLY THE TO DATE IS DEFINED

All data up until the defined end date. No start date defined.

Example:

Concept: Acute Myocardial Infarction

From date:

To date: 12/31/2005

In this example, if a patient has a diagnosis of Acute Myocardial Infarction (MI) entered on the record prior to and including 12/31/2005 then their data will be included. If the diagnosis was entered after 12/31/2005 then it will not be included.

Numeric Value

The numeric value is defined in the source system when the result is entered.  The following are some example formats that can be found in the i2b2 Demo database.

§         200

§         1.3

 

Unlike flag and text values, numeric values use operators, such as greater than or less than when setting the constraints. The following is a listing of operators currently in use.

 

Operator

Description/Use

Less Than (<)

Any value less than the number entered.

Less Than or Equal To (<=)

Any value less than or equal to the number entered (inclusive).

Equal To (=)

An exact match to the number entered.

Between

Any value in the range of numbers entered.

Greater Than (>)

Any value greater than the number entered.

Greater Than or Equal To (>=)

Any value greater than or equal to the number entered (inclusive).
