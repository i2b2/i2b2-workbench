Introduction

This section describes the layout and contents of the Patient Mapping view.

 

Layout of the Patient Mapping View

The Patient Mapping view allows users to see the site ids (MRNs) for a group of patients.

 

 

 

The Patient Set Box is where you can drop the patient set you want to display in the Patient Table section.

 

 

Navigation

This section contains information about setting the display options.

 

Display Options

The patient mapping information is either stored in the Data Repository Cell (CRC) or the Identity Management Cell (IM).  By default the Patient Mapping view will send a request message to the Data Repository Cell (CRC). For those sites that are using the Identity Management Cell (IM) they have the option to change this default setting. The setting is defined in Patient Mapping Options, which can be accessed by clicking on the Options button ().

 

In Patient Mapping Options, users also have the ability to define how many patients are to be loaded into the table at any one time. Both of these options are defined in the sections that follow this one.

 

 

 

 

NOTE:

 

 

Please see your site administrator if you do not know whether or not your patient mapping data is stored in the Data Repository Cell (CRC) or the Identity Management Cell (IM).

 

 

 

Send Requests to the IM Cell

A check mark next to Send PDO requests to IM cell means the Patient Mapping view will send the request message to the IM cell instead of the CRC cell.

 

 

IMPORTANT:

 

 

All settings in Patient Mapping Options are specific to your session. Therefore if you log out of the i2b2 Workbench you will need to check this off again if you wish to communicate with the IM cell.

 

 

 

Paging

The value entered at Increment Size field in the Patient Mapping options will determine how many patients will be displayed in the Patient Mapping view. This value can be changed at any time.

 

View Patient Mappings

This section contains information about viewing the site mappings for patient sets added to the Patient Mapping view.

 

Draggable Items

In order to view the patient mappings you will need to add either a set of patients or an individual patient to the Patient Mapping view.  Both of these items can be dragged from other i2b2 Workbench views and dropped (copied) into the Patient Mapping view.

 

1.       Patient Sets

A specific set of patients can be used to view the site ids (MRNs) by dragging the Patient set () from one of the following views:

·         Previous Queries view

·         Patient Sets view

·         Workplace view

 

 

2.       Patient

An individual patient can be used to view the site ids (MRNs) by dragging the patient () from one of the following views:

·         Previous Queries view

·         Patient Sets view

·         Workplace view

 

 

Add Patient Sets

As stated earlier, Patient sets can be added from one of the following views:

·         Previous Queries view

·         Patient Sets view

·         Workplace view

 

How to add the patient set from each of these views is explained in the next few sections.

 

Previous Queries View

The Previous Queries view displays the queries that were run by the user.  Within the previous query there is a result and provided Patient set or Timeline were checked off when the query was run there will be a patient set associated with the previous query.

 

1.       In the Previous Queries view click on the plus sign () next to the name of the previous query that contains the patient set.

2.       Click on the plus sign () next to the results folder ().

3.       Highlight the Patient set () by clicking on its name.

4.       While holding the left mouse button down, drag the item over to the Patient Mapping view.

5.       Drop the patient set at Patient Set box, which is labeled Patient Set Identifier.

6.       The list of patients and all their site ids (MRNs) will display in the patient table.

Saving Patient Mappings

In the Patient Mapping view there are two methods of saving the patient mappings to an external file.  Users can either automatically or manually save it.

 

Automatically Save

Users have the ability to automatically save the patient mappings to an external file by checking off save patients to a file automatically before you drop the patient set into the view.  As long as it is checked off you will be prompted with the Save as window when you drop the patient set into the view.

 

 

WARNING:

 

 

You need to make sure you have checked off save patients to a file automatically BEFORE you drop the patient set into the view.

 

 

 

Manually Save

Users can choose to manually save the patient mappings by clicking on the Save button located at the bottom of the view. As soon as you click on this button the Save as window will display. You can select where to save the file and change the default name of the file.

 

 

Open Patient Mapping file

In the Patient Mapping view you can choose to view an external csv file that contains your patient mapping information.

 

1.       Click on the Open Patient Set button located at the bottom of the view.

2.       A select file window will open.

3.       Find the file you wish to view in the Patient Mapping view.

4.       The patient mapping information will display in the view.

 

 

Encryption

This section contains information about the data encryption key and converting site IDs (MRNs) to be either encrypted or unencrypted.

 

Decryption Key

Depending on how your data is stored in your database you may be asked for a Decryption key when viewing a patient mapping file.  If your site ids (MRNs) are encrypted then you will need to have the appropriate key to view all the MRN’s. If you do not have the correct key the ids will display in the patient mapping view as encrypted.

 

 

 

Entering Decryption Key

When you drag over a patient set, the system will check to see if any of the patients in the set have encrypted data. If they do then you will be prompted with the following window.

 

 

 

If you choose to manually type in the key you will be prompted with the input window.

 

 

Provided you have entered the key correctly the patient mappings will appear in the view decrypted.

 

 

Converting Site IDs (MRNs)

The Patient Mapping view provides a tool to either encrypt all your site ids or decrypt them. This tool is accessed by clicking on the Convert Between Patient Sets button.

 

 

 

Input and Output Files

The Input File is the external file that currently exists; it contains either the encrypted or unencrypted site IDs (MRNs).

 

The Output File is the external file that will be generated when the site IDs (MRNs) are converted.

 

File Formats

There are currently several different options available.

 

Two Columns – Patient Number and Unencrypted MRN

This file format contains only the patient number and the site ID (MRN). The site ID is not encrypted.

 

Three Columns – Patient Number, Site Name and Unencrypted MRN

This file format contains patient number, the name of the site associated with the site ID, and the site ID (MRN). The site ID is not encrypted.

 

Three Columns with Hive number row Unencrypted MRN

This file format contains patient number, the name of the site associated with the site ID, and the site ID (MRN). The site ID is not encrypted.

 

This format is only available for Output files.

 

 

IMPORTANT:

 

 

This output format will create a new row for each site id. This new row will have the site of HIVE and the IDs associated will be the patient number for each existing record.

 

 

 

Two Columns – Patient Number and encrypted MRN

This file format contains only the patient number and the site ID (MRN). The site ID is encrypted.

 

Three Columns – Patient Number, Site Name and encrypted MRN

This file format contains patient number, the name of the site associated with the site ID, and the site ID (MRN). The site ID is encrypted.

 

Three Columns with Hive number row encrypted MRN

This file format contains patient number, the name of the site associated with the site ID, and the site ID (MRN). The site ID is encrypted.

 

This format is only available for Output files.

 

 

IMPORTANT:

 

 

This output format will create a new row for each site id. This new row will have the site of HIVE and the IDs associated will be the patient number for each existing record.

 

 

 

Ten Columns (i2b2 1.6 Patient Mapping) Unencrypted MRN

This file format contains all the columns in the 1.6 version of the Patient Mapping table located in the i2b2 Database. The site ID is not encrypted.

 

This format is only available for Input files.

 

Eleven Columns (i2b2 1.7 Patient Mapping) Unencrypted MRN

This file format contains all the columns in the 1.7 version of the Patient Mapping table located in the i2b2 Database. The site ID is not encrypted.

 

This format is only available for Input files.

 

 

Ten Columns (i2b2 1.6 Patient Mapping) encrypted MRN

This file format contains all the columns in the 1.6 version of the Patient Mapping table located in the i2b2 Database. The site ID is encrypted.

 

This format is only available for Output files when the input file format is Ten Columns (i2b2 1.6 Patient Mapping) Unencrypted MRN.

 

Eleven Columns (i2b2 1.7 Patient Mapping) encrypted MRN

This file format contains all the columns in the 1.7 version of the Patient Mapping table located in the i2b2 Database. The site ID is encrypted.

 

This format is only available for Output files when the input file format is Eleven Columns (i2b2 1.7 Patient Mapping) Unencrypted MRN.
