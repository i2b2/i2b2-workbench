Introduction

The i2b2 Workbench is a collection of views that communicate with the i2b2 Cells and help aggregate their functionality in the Hive. Each view is designed to interact with a hive cell; collectively these views provide a cohesive entity to tie all the i2b2 cells together and ultimately make it easier for clinical investigators to gather and analyze clinical information.

Application Title Bar

The Application Title Bar contains the name of the application (i2b2 Workbench) as well as the following buttons.

 

Minimize

The Minimize button () will remove the window from view. The window is still active and can be restored by clicking on the application name located on the windows taskbar.

 

Maximize

The Maximize button () will make the window fill the screen.

 

Restore Down

The Restore Down button () will resize the window so it no longer fills the screen.  You can further change the size of the window by using your mouse to drag the sides of the window in or out.

 

Close

The Close button () will exit the window and close the application.

 

Warning: if you exit the window you will need to log back in the i2b2 Workbench, there is no way to “restore” the window.

Add new features

The “Add new features” is an install wizard that will walk the user through adding optional features (plug-ins) to the i2b2 Workbench.

 

How To Add A New Feature

The following steps outline how to add a new feature to the i2b2 Workbench.

 

1.       Click on the Help Menu.

2.       Select Add new features from the menu.

3.       A new window will open displaying the Search Results page of the install.

 

Figure 20       Feature Install: Search/Selection Page

 

 

4.       Click on the box next to the feature(s) you want to install (checkmark will appear).

 

Figure 21       Feature Selection

 

 

5.       Click on the Next button located at the bottom of the page.

6.       The Feature License page will display.

 

Figure 22       Feature License Agreement

 

 

7.       After reviewing the license click on the radial button next to “I accept the terms in the license agreement”.

8.       Click on the Next button.

9.       The Installation page will display with all the features you have selected to install.

 

Figure 23       Feature Installation Page

 

 

10.   By default the feature will be installed in the Feature folder located in the same folder as the i2b2 Workbench.  This default location can be changed by clicking on the Change Location button.

11.   Click on the Finish Button.

12.   The Feature Verification page will display.

13.   Click on either Install or Install All button.

14.   Once the feature has been installed an update message will display.

 

Figure 24       Feature Install/Update Message

 

 

15.   Click on Yes to restart the workbench.

16.   The workbench will close and bring you back to the Login page.

17.   Log into the workbench.

18.   The views for the features you just added can be found by going to the Show View Menu and clicking on Other.

 

Figure 25       Show View: Other

 

 

 

i2b2 Perspectives and Views

 

i2b2 Perspectives

Each workbench can contain one or many perspectives.  A perspective defines the initial set and layout of views for the i2b2 Workbench.

 

i2b2 Views

The i2b2 views are individual widgets designed to support features developed for the i2b2 workbench. A default set of views appear when a user logs in to the workbench for the first time.  The views can be removed or moved within the workbench.

 

 

 

Navigation

 

Moving a View

 

1.       Click on the tab of the view you want to move.

2.       While holding down the left mouse button, drag the view to a new area of the desktop.

3.       Once the area displays a grid line you can drop the view into that area.

Note: you may need to drag the view to the edge of the section you want to move it to.

 

4.       The view will now appear in the new location.

 

Adding a View

 

1.       Open the Window menu located on the Application Menu Bar.

2.       Select Show View.

3.       A list of the core i2b2 views will appear.

4.       Select the view you want to open.

Note: if the view does not appear in the list click on “other” which is located at the bottom of the menu.  This will open an expanded list of views.  Views for optional features will be listed here.

 

 

Removing a View

 

1.       Click on the view you want to remove.

2.       Click on the Close Button () located in the top right corner of the view.

3.       The view will no longer appear in the desktop.

 

Maximize/Minimize/Restore Views

All of the i2b2 views can be maximized to take up the entire desktop area, minimized so it remains active but not visible on the desktop, and restored to its original size and location.

 

Maximize a View

 

1.       Click on the view you want to maximize.

2.       Click on the Maximize Button () located in the top right corner of the view.

3.       The view will now be expanded to fill the desktop area.

 

Figure 26       Expanded View

 

 

Note: the other views have been minimized and appear on the task bars located on the left, right and bottom of the desktop area.

 

 

Minimize a View

 

1.       Click on the view you want to minimize.

2.       Click on the Minimze Button () located in the top right corner of the view.

3.       The view will no longer appear in the desktop area.

4.       A taskbar will appear either on the left, right or bottom of the desktop area.

5.       The restore button and an icon will appear in the taskbar for each view that was minimized.

 

Figure 27       Minimize View Information in Taskbar

 

 

6.       Using your mouse, hover over the icon to display the name of the view that is minimized.

 

Figure 28       Display Name of Minimized View

 

 

Restore a View

There are two reasons in which a view may need to be restored. One is because the view was maximized and the other is because the view was minimized.  Either way the user wants to bring the view back to its original size and location on the desktop.

 

Restore a Maximized View

1.       Click on the Restore Button () located in the top right corner of the view.

2.       The view will now be restored to its original size and location in the desktop area.

 

Restore a Minimized View

 

1.       A minimized view will appear in a taskbar located on either the left, right or bottom side of the desktop area.

2.       Using your mouse, hover over the icon () in the taskbar.

3.       The name of the minimized view will be displayed.

4.       Click on the Restore Button () located next to the icon.

5.       The view will now be restored to its original size and location in the desktop area.

 

 

 

Options

Options allow users to set how information in a view is retrieved or displayed. These settings are specific to the view and may not be available for all views. If it is available the Option button () will appear in the top right corner of the view.

 

Note: additional information on how to use options can be found in the documentation for each view.

 

 

Search

The search feature allows users to find specific information in a view. This feature is not available to all views. If it is available the Search button () will appear in the top right corner of the view.

 

Note: additional information on how to use the search feature can be found in the documentation for each view.

 

XML Messages

Messages are sent to and from the i2b2 database using Extensible Markup Language (XML).  The messages are specific to the view and can be seen by clicking on the XML Messages button () located in the top right corner of the view.  Viewing the messages can help when troubleshooting issues that may arise.

 

The XML messages will be stored in the log for each session.  The entries will be labeled with either Sent or Received and the time the communication occurred.

 

Sent Messages are sent from the i2b2 Workbench to the database.

Received Messages are sent to the i2b2 Workbench from the database.
